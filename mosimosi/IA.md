# 여보세요 — Information Architecture (IA)

---

## 1. IA 설계 원칙

1. **홈 = 게임 로비**: 훈련 앱의 코스 목록이 아니라 게임 모드 선택. 진입 즉시 "보스전 / 배틀"이 지배적.
2. **두 폼팩터는 동등한 게임 클라이언트**: 데스크톱도 모든 화면·모드를 완전 지원. 폼팩터 차이는 레이아웃 차이일 뿐, 기능 삭제가 아님.
3. **통화 화면은 막다른 방**: 통화 중 네비게이션 제거, 이탈은 "끊기"로만. 배틀 실수 이탈 방지.
4. **비밀 정보는 화면 계층에서 분리**: 배틀 비밀 목표·규정 카드는 프라이빗 패널. 관전 스트림엔 서버가 원천 제외.
5. **결과 → 재도전 루프 최단화**: 재도전/다음 보스/재매칭 1탭.
6. **리포트는 보상 화면에 내장**: 별도 학습 섹션이 아니라 결과 화면 하위 탭("전리품").

---

## 2. 폼팩터별 레이아웃 규칙 (핵심)

화면을 두 부류로 나눠 데스크톱 레이아웃을 결정한다:

| 화면 성격 | 모바일 | 데스크톱 |
|---|---|---|
| 세로 단일 흐름 (통화) | 세로 전화 UI | **중앙 고정폭 폰 목업 + 양옆 게임 HUD 패널** |
| 넓게 펴지는 것 (홈·도감·전적·대시보드) | 세로 스크롤/2열 | 네이티브 와이드(다열 그리드) — 폰 목업 금지 |

**데스크톱 통화 화면(중앙 폰 + HUD) 구성**
- 상단(전체 폭): 모멘텀 게이지 + 양측 이름표 + 타이머
- 중앙(고정폭 ~300px 폰 목업): 발신자·실시간 자막·마이크/끊기 — 모바일 통화 위젯 재사용
- 좌측 패널: (배틀) 내 비밀 목표 / 규정 카드 / 이벤트 타임라인 · (싱글) 클리어 조건 체크리스트
- 우측 패널: AI 코치 귓속말 로그 / 실시간 자막 전체 히스토리
- 이 레이아웃은 관전 화면의 부모(중앙 폰 2개 + 가운데 게이지·캐스터)

│   ├── 0.1 서비스 소개 (콘셉트/보스전/배틀)
│   ├── 0.2 마이크 권한 + STT 준비 (데스크톱: 서버 STT 연결 확인)
│   └── 0.3 닉네임 설정
│
├── 1. 홈 (게임 로비)                          [GNB: 홈]
│   ├── → 2. 보스전 · → 3. 전화 배틀
│   ├── 이어하기 카드 (마지막 도전 보스)
│   └── 오늘의 전적 요약
│   └── (데스크톱) 와이드 로비 — 모드 카드 + 최근 활동 병렬
│
├── 2. 보스전 (싱글) — 두 폼팩터 완전 지원
│   ├── 2.1 보스 도감                          [GNB: 도감]
│   │   ├── (모바일) 2열 카드 · (데스크톱) 4~5열 그리드
│   │   └── 카드 상태: 잠김 / 해금 / 클리어(최고점)
│   ├── 2.2 브리핑 (시나리오·클리어 조건·제한 시간·최고 기록)
│   │   └── [발신] → 2.3
│   ├── 2.3 통화 화면 ★막다른 방
│   │   ├── (모바일) 세로 전화 UI
│   │   ├── (데스크톱) 중앙 폰 + 좌:클리어 조건 / 우:AI 코치·자막 로그
│   │   ├── 공통: 인내심 게이지, 이벤트 팝업, Push-to-talk, 끊기

│       ├── 탭B 리포트 (더듬 구간·개선 스크립트·타임라인)
│       ├── 하이라이트 공유 카드
│       └── CTA: 재도전 / 다음 보스 / 도감
│
├── 3. 전화 배틀 (멀티) — 두 폼팩터 완전 지원, 크로스 폼팩터 매칭
│   ├── 3.1 매칭 대기
│   │   ├── 검색중(경과) / 매칭성사(상대·역할, 상대 폼팩터 표시 가능)
│   │   └── 30초 폴백: "AI 상담원과 배틀할까요?"
│   ├── 3.2 비공개 브리핑 ★프라이빗
│   │   ├── 공통: 상황·내 역할
│   │   ├── 비밀: 내 목표 / (상담원) 규정 카드

│   │   ├── 상단: 모멘텀 게이지(줄다리기, 양측 공통)
│   │   ├── (모바일) 세로 + 접이식 비밀 목표 칩 + AI 코치 접이식

│   │   └── B-lite: 상대 발화 TTS 재생
│   ├── 3.4 판정 대기 (심판 로딩 연출)
│   └── 3.5 배틀 결과

│       ├── 탭B 비밀 공개 (서로의 목표·규정 상호 공개 — 연출 이벤트)

│       └── CTA: 재매칭 / 홈
│
├── 4. 내 전적                                 [GNB: 전적]
│   ├── 4.1 대시보드 (점수·필러·침묵 추이 — 데스크톱 와이드)
│   ├── 4.2 전적 목록 (싱글/배틀 필터)
│   │   └── 4.2.1 판 상세 (Whisper 정제 트랜스크립트 리플레이 + 당시 리포트)

│
├── 5. 관전 모드 (데스크톱 우선, 데모/공유용)
│   ├── 진입: 배틀 room 관전 (GNB 미노출, 링크/코드 진입)

└── 6. 설정 (프로필 → 시트)

    ├── 마이크 테스트 (데스크톱: 서버 STT 연결 상태 + 텍스트 입력 폴백 토글)
    ├── 음성 설정 (TTS 속도, 자동 턴 감지 on/off)
    └── 데이터 안내 (음성 미저장 고지)

## 4. 글로벌 네비게이션

| 탭 | 진입 | 노출 |
|---|---|---|
| 홈 | 1 로비 | 항상 |
| 도감 | 2.1 도감 | 항상 |
| 전적 | 4.1 대시보드 | 항상 |

- 모바일: 하단 탭바 3개 + 우상단 프로필
- 데스크톱: 좌측 사이드 레일 또는 상단 바(넓은 화면 관례) — 동일 3개 목적지
- GNB 숨김: 2.3 / 3.2 / 3.3 / 3.4
- 배틀(3.x)은 GNB 탭이 아니라 홈 CTA로만 진입(실수 매칭 방지)
- 관전(5)은 데모 맥락 진입(GNB 미노출)

## 5. 핵심 사용자 플로우

### F1. 첫 방문 → 첫 클리어
0.1 소개 → 0.2 권한 → 0.3 닉네임 → 1 홈 → 보스#1 치킨집 → 2.2 → 2.3 → 2.4 "다음 보스 해금!" → 도감
- 첫 세션 반드시 쉬운 클리어 1회 경험. 보스#1 실패 조건 느슨하게.

### F2. 보스전 반복 루프
도감 → 브리핑 → 통화 → 결과(재도전|다음) → 도감. 재도전은 브리핑 스킵.

### F3. 배틀 풀 플로우 (크로스 폼팩터)
홈 → 3.1 매칭(모바일↔데스크톱 허용) → 3.2 비공개 브리핑 → 3.3 통화 → 3.4 판정 → 3.5 결과 → 재매칭|홈
- 30초 경과 시 AI 폴백. 상대 연결 15초 끊김 → "AI 이어받기 / 판 무효" 모달.

### F4. 관전 플로우 (데모)
배틀 room 생성 → 데스크톱에서 관전 진입 → 게이지·캐스터로 중계 → 종료 시 판정 함께 노출

### F5. 리포트 회고
전적 → 판 상세 → 트랜스크립트 하이라이트 탭 → 공유 카드 재생성

## 6. 화면별 콘텐츠 인벤토리 (우선순위)

### 2.3 통화 화면 (싱글)
1. Push-to-talk 버튼 (엄지 존, 최대 비중)
2. 타이머 + 인내심 게이지 (상단)
3. 실시간 자막 (모바일: 직전 2~3발화 / 데스크톱: 우측 전체 로그)
4. 이벤트 팝업 (오버레이, 2초 소멸)
5. 클리어 조건 진행 (모바일: 접이식 칩 / 데스크톱: 좌측 상시)
6. 끊기 (접근성 낮춤)

### 3.3 배틀 통화 화면 (싱글과 차이)
- 인내심 게이지 → 모멘텀 게이지(좌우 줄다리기)
- 비밀 목표(모바일 칩/데스크톱 좌측), AI 코치(모바일 접이식/데스크톱 우측 로그)
- Push-to-talk 대신 자유 대화 + 마이크 활성 인디케이터

### 3.5 배틀 결과 (탭 순서 근거)
- A 판정 디폴트(승패 먼저) → B 비밀 공개(연출 이벤트로 분리) → C 리포트(감정 소화 후 학습)

## 10. 미해결 IA 결정사항
1. 데스크톱 GNB: 좌측 레일 vs 상단 바
2. 배틀 재대결(같은 상대) 지원 여부 → 3.5 CTA에 영향
3. 도감 정렬: 난이도순 vs 생활/직장 테마
4. 관전 진입 방식: 데모 전용 코드 vs 공유 기능 승격
5. 폰 목업 표현: 사실적 기기 프레임 vs 은유만 남긴 둥근 카드
6. 이어하기 카드 vs 데일리 미션 — 홈 두 번째 위젯
