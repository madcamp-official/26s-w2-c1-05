# 인수인계: 보스 프롬프트 개편 + 신규 보스 추가

너는 "여보세요"(mosimosi) — Flutter 앱 + FastAPI 서버로 된 전화 연습 게임 — 의 보스
페르소나를 개편·추가하는 작업을 맡았다. 아래를 **끝까지 읽고** 시작해라. 이 문서는
직전 세션이 방금 구축한 **감정 태그 파이프라인**을 전제로 한다.

---

## 0. 먼저 이해할 것 — 보스 대사가 생성되는 구조

```
[통화 시작]
  보스가 먼저 받는다: call_session.dart가 seedUser로 _bossReply 호출
       ↓
[매 턴]
  유저 발화(STT) → 최근 8발화 + 시스템 프롬프트 → 게임서버 /llm/chat (SSE 스트리밍)
       ↓                                    └ task=boss_turn → vLLM(Qwen3-14B-AWQ)
  스트리밍 델타 → 문장부호 단위로 잘라 → /tts/synthesize → Qwen3-TTS 재생
```

- **시스템 프롬프트 = `boss.buildSystemPrompt(variables)`** (mosimosi/lib/core/models/boss.dart):
  `personaPrompt` + 난이도 텍스트 + `[이번 판 상황 변수]`(랜덤) + `bossCommonRules` 를 합침.
- **랜덤성(신선함)**: 판 시작 시 Gemini가 `boss.name`+`boss.scenario`만 보고 상황 변수
  2~3개를 생성해 `[이번 판 상황 변수 — 대화 중 자연스럽게 드러내라]` 블록에 주입.
  (mosimosi/lib/core/call/llm_tasks.dart의 generateScenarioVariables)

## 1. ★감정 태그 파이프라인 (직전 세션 신규 구축 — 반드시 활용)

보스는 **매 응답 맨 앞에 감정을 대괄호 하나로 붙인다**. 클라가 그 태그를 떼어
TTS에 감정으로 넘기고(자막엔 안 보임), Qwen이 같은 화자·음색을 유지하며 그 순간의
감정 톤으로 합성한다.

- **감정 6종 (이 외의 값은 무시됨)**: `평온` `상냥` `짜증` `분노` `미안` `당황`
- **출력 형식**: `[짜증] 아니 그 시간은 안 된다고 말씀드렸잖아요.`
- 이 규칙은 **`bossCommonRules`(boss.dart)에 이미 들어 있어 모든 보스에 공통 적용**된다.
  즉 personaPrompt에서 감정 태그 형식을 다시 설명할 필요 없다. 대신 **"언제 어떤
  감정을 쓰라"는 트리거를 personaPrompt/few-shot으로 지도**하면 된다.
  예: "평소엔 [상냥], 손님이 규정을 어기라 하면 [분노], 못 알아들으면 [짜증]."

**감정→목소리 매핑은 서버 tts.py의 `_EMOTION_INSTRUCTION`에 있다** (건드릴 필요 없음).
너의 일은 "보스가 맥락에 맞는 감정 태그를 뱉게" 프롬프트로 유도하는 것뿐이다.

## 2. 어디를 수정하나 — 파일별 지도

| 목적 | 파일 | 무엇을 |
|---|---|---|
| **보스 성격·대사·시나리오** | `mosimosi/lib/core/data/bosses.dart` | 각 Boss의 `personaPrompt`, `scenario`, `clearConditions`, `difficulty` |
| 공통 규칙(감정 태그 등) | `mosimosi/lib/core/models/boss.dart` `bossCommonRules` | **건드리지 마라** (이미 완성). Boss 모델 필드는 여기 |
| 보스별 목소리(화자) | `server/game/app/tts.py` `_QWEN_VOICE_MAP` | 신규 보스의 `voice_name`→Qwen 화자 매핑 추가 시 |
| 도감 목록 표시 | `mosimosi/lib/features/bosses/boss_list_screen.dart` `_entries` | 신규 보스를 도감에 노출할 때 |

### 오프닝 대사(고정 첫 마디)를 정하는 법 — 중요

현재 보스의 **첫 대사는 LLM이 생성**한다 (call_session.dart:146이 `seedUser: '(따르릉—
전화가 연결되었다...)'`로 _bossReply를 킥). 고정 오프닝을 원하면 **personaPrompt 안에
강하게 못박아라**:

```
[통화 시작 규칙]
- 네 첫 대사는 반드시 아래 문장으로 시작한다 (토씨 하나 바꾸지 마라):
  "[상냥] 안녕하세요 무던치킨입니다! 지금 주문이 많이 밀려있네요, 잠시만요… 네 고객님! 어떤 메뉴 주문하시겠어요?"
```

- 오프닝에도 **감정 태그를 붙여야** 한다 (맨 앞 `[상냥]` 등).
- "2초 침묵" 같은 연출은 말줄임표(`…`)로 표현하면 TTS가 호흡을 준다. 실제 정확한
  침묵 타이밍 제어는 지금 파이프라인에 없으니 `…`로 근사하라.

## 3. 두 가지 어려운 요구 — 반드시 이렇게 처리하라

### (A) "스케줄 모순 없음" (치과 예약 등)

LLM은 상태 일관성에 약하고, 히스토리가 최근 8발화뿐이라 앞서 "3시 안 돼요"라고 한 걸
잊고 나중에 "3시 어떠세요?"라고 모순낼 수 있다. **해결: 가능/불가 시간표를
personaPrompt에 상수로 명시**해서 LLM이 그 표를 읽고만 답하게 하라.

```
[오늘 예약 현황 — 이 표를 절대 어기지 마라]
- 예약 가능: 오후 2시, 오후 4시 30분
- 예약 불가(꽉 참): 오전 전체, 오후 3시, 오후 5시
- 손님이 불가 시간을 요청하면 거절하고, 위 '가능' 시간 중 하나를 역제안해라.
- 이미 불가라고 말한 시간을 나중에 가능하다고 하면 절대 안 된다.
```

판마다 다르게 하고 싶으면 이 표를 **상황 변수 생성 프롬프트로 뽑을 수도 있으나**(llm_tasks.dart),
데모 안정성을 위해선 **보스에 고정 표를 박는 것**을 우선 추천한다. (변수 생성은
비결정적이라 모순 위험이 다시 생김.)

### (B) "품절 메뉴 랜덤" (치킨) 같은 판별 요소

두 가지 방법:
1. **상황 변수 활용** (권장, 이미 있는 메커니즘): 판 시작 시 Gemini가 "오늘 품절: 양념치킨"
   같은 변수를 생성 → `[이번 판 상황 변수]`로 주입됨. personaPrompt에 "품절 메뉴를
   손님이 주문하면 [미안] 거절하고 다른 메뉴를 권해라"를 넣으면 연결된다.
2. **고정**: personaPrompt에 품절 메뉴를 직접 박으면 매판 같음(재미 감소).
   → 1번을 쓰되, 만약 상황 변수가 품절 메뉴를 안 뽑으면 대비해 "품절 메뉴가 지정되지
   않았으면 자유롭게 하나 정해라"를 추가.

## 4. personaPrompt 작성 규칙 (기존 3보스 참고)

`bosses.dart`의 기존 보스가 좋은 템플릿이다. 구성:
1. **정체·성격·말투** 2~3줄
2. **역할**(무엇을 받아내/거절해야 하는가) + 조건부 행동(언제 물러서나)
3. **few-shot 예시 2~3개** — 실제 대화 흐름을 보여주는 게 가장 강력. **예시에도 감정
   태그를 붙여라** (모델이 형식을 학습한다).
4. 감정 트리거 지도 (언제 어떤 감정)

주의:
- `difficulty`(DifficultyParams): maxSentences(1~2), cooperativeness(1~5),
  surpriseFreq(1~5), interrupts(bool) — 난이도 텍스트로 자동 변환됨. 캐릭터에 맞게 조정.
- `clearConditions`: 최종 심판이 이걸로 채점하니 **측정 가능하게** 써라.
- 응답은 **1~2문장 짧게** (maxSentences). 긴 설명 금지 — 전화 통화답게.
- 실존 업체·인물·전화번호 금지 (bossCommonRules가 강제하지만 예시에서도 지켜라).

## 5. 작업 대상 — 보스 6종

### 기존 3종 수정 (bosses.dart의 chicken/dental/refund)

**치킨집 사장 (chicken)** — 화자 유지(Charon→uncle_fu)
- 60대 남성, 바쁘지만 밝고 호탕한 성격 (기본 감정 [상냥]/[평온], 서두를 땐 빠르게)
- 오프닝 고정: "[상냥] 안녕하세요 OO치킨입니다! 지금 주문이 많이 밀려있네요, 잠시만요…
  네 고객님! 어떤 메뉴 주문하시겠어요?"
- 유저는 메뉴 주문 + **주소까지** 말해야 클리어. 주소 안 말하면 [상냥] 되물어라.
- 품절 메뉴(상황 변수)면 [미안] 거절 + 다른 메뉴 권유.
- 랜덤 이벤트: 치킨무 빼드릴지, 리뷰 이벤트 부탁 등 (few-shot에 1개 예시).
- clearConditions: 메뉴 전달 / 주소 전달 / (품절 대응 시 대체 주문)

**치과 접수원 (dental)** — 화자 유지(Kore→vivian)
- 40대 여성, 업무 많아 살짝 짜증. 기본 [평온](사무적 비즈니스 상냥), 트리거 시 [짜증].
- 오프닝 고정: "[평온] 네~ OO치과입니다~ 무슨 일로 연락 주셨어요?"
- 유저가 충치 설명 + 예약 시도. **스케줄 모순 금지 → 위 3(A)의 고정 시간표 필수.**
- 자꾸 불가 시간만 말하면 가능 시간 역제안 (표 안에서만).
- 유저가 말 느리거나 못 알아들으면 [짜증] 추가 → "손님이 3초 이상 머뭇거리거나
  같은 걸 되물으면 [짜증]으로 전환" 같은 트리거를 넣어라.
- "시술이 아니라 충치(진료) 예약"임을 알면 [짜증]으로 태도 나빠짐.
- 은연중 업무 많음 노출 ("아우 오늘 대기가 많아서…").

**환불 상담원 (refund)** — 화자 유지(Aoede→sohee)
- 30대 여성, 환불 거절했는데 계속 조르면 [평온]→[짜증]→[분노]로 **점증**.
- 오프닝: "[평온] 네, 무엇을 도와드릴까요?" **만.** (고객이 환불 얘기 꺼내기 전에
  먼저 환불 얘기를 꺼내면 안 됨 — 이걸 규칙으로 명시.)
- 환불 얘기 나오면 개봉 여부 물음 (이미 개봉했다 말했으면 생략, 되물을 수도 있음).
- 개봉했으면 환불 거부: "[짜증] 홈페이지에 나와 있어요, 확인해 보세요." 식으로
  귀찮아하며 뭉뚱그림.
- 고객이 **구체적 근거**(홈페이지 문구는 법적 효력 없다 등)를 대면: 처음엔 [짜증]
  얼버무리다 → 계속 논리적이면 [미안]하게 인정하고 들어줌.
- 감정 점증이 이 보스의 핵심 — few-shot으로 단계별 변화를 보여줘라.

### 신규 3종 추가 (bosses.dart에 Boss 항목 신설 + 아래 부수 작업)

신규 보스는 **id 새로 정하고**(professor_grade, professor_grad, parttime 등), 아래를
함께 해야 도감/목소리가 연결된다:
1. `bosses.dart` — Boss 항목 추가 (personaPrompt 등 full)
2. `server/game/app/tts.py` `_QWEN_VOICE_MAP` — 새 voice_name → Qwen 화자 매핑 추가
   (기존 화자 재사용 가능: 남성=uncle_fu, 여성=vivian/sohee. 화자 목록: aiden, dylan,
   eric, ono_anna, ryan, serena, sohee, uncle_fu, vivian). **voice_name은
   `ko-KR-Chirp3-HD-XXX` 형식**이고 이 문자열이 클라 preset과 서버 map의 키를 잇는다.
   신규는 기존에 안 쓴 Chirp3 이름(Puck/Fenrir/Orus/Leda/Zephyr 등)을 골라 map에 등록.
3. `boss_list_screen.dart` `_entries` — 도감 노출 시 (번호 3~7 스텁 자리 활용)
4. Boss 모델의 `voicePreset: TtsVoicePreset(voiceName: '...')` 지정 필수

**교수1·성적 (professor_grade)** — 60대 남성 (uncle_fu 등)
- 처음 정중 → 나중에 근엄·엄격. [평온]→[분노]는 아니고 [평온]→근엄한 [짜증] 정도.
- 오프닝: "[평온] 안녕하세요, OOO 교수입니다. 학생, 무슨 일로 전화했나요?"
  (학생이 성적 얘기 꺼내기 전에 먼저 꺼내지 말 것.)
- 성적 얘기 들으면 "점수는 충분히 잘 줬다"고 방어.
- 유저가 "시험/과제 점수 좋다" 하면 "확인해보니 출결 문제가 있었다"로 반격.
- 유저가 논리적 이유(결석한 날 아팠다 등) 대면 토론하다 납득 → 점수 올려줌.
- clearConditions: 논리적 근거 제시 / 교수 반박 통과 / 점수 상향 확답.

**교수2·대학원 (professor_grad)** — 60대 남성 (다른 남성 화자로 구분: dylan/eric 등)
- 밝고 긍정적, 제안을 쉽사리 거절 못 하게 만드는 느낌. 기본 [상냥].
- 오프닝: "[상냥] OOO 학생, 안녕하세요! 이번에 몰입캠프 과목에서 아주 우수한 성적을
  받았더라고요! 이렇게 뛰어난 학생 오랜만이네요!"
- 안부인사 하다 대학원 영입 시도. 유저는 이유 대며 회피 → 적당히 둘러대면 포기,
  그때 [미안]하게 시무룩.
- clearConditions: 대학원 제안 거절 성공 / 관계 안 망치고 마무리.

**알바 사장 (parttime)** — 30~40대 남성 (ryan 등)
- 알바 위하는 척 [상냥]/[미안]하지만 실은 시급 인상 회피(주제 돌리기).
- 오프닝: "[평온] 네, 무슨 일로 전화했어요?"
- 유저가 시급 인상 요구. **근거 잘 대면** 내키지 않지만 [미안] 수용. **근거 없이
  그냥 올려달라 하면** [분노].
- clearConditions: 인상 근거 제시 / 사장 회피 돌파 / 인상 확답.

## 6. 검증 (반드시 수행)

1. `cd mosimosi && flutter analyze` — 0 issues.
2. `flutter test` — 전부 통과.
3. **실 LLM e2e** (선택이지만 권장): 서버가 camp-3에서 돌고 있다(터널
   `https://graceheeseo.madcamp-kaist.org`). 보스 대사가 감정 태그를 붙여 나오는지
   확인하려면 `/llm/chat`에 boss_turn 요청을 직접 쏘거나, 앱을 빌드해 통화해봐라.
   특히 **오프닝 고정 대사가 정확히 나오는지**, **감정 태그가 자막에 안 새는지**
   (call_session의 _consumeEmotionTag가 떼어냄) 확인.
4. Boss 모델 필드(voicePreset 등)를 빠뜨리면 컴파일 에러가 난다 — analyze로 잡힌다.

## 7. 배포 (서버 파일을 고쳤을 때만)

`bosses.dart`만 고쳤으면 **앱 재빌드만** 하면 된다(서버 무관). `tts.py`를 고쳤으면
camp-3에 반영해야 한다:
- 로컬 `server/`와 camp-3 `/root/server`는 **항상 동일하게 유지**하는 규칙이 있다.
- `scp server/game/app/tts.py MadCamp:/root/server/game/app/tts.py` 후
  `ssh MadCamp 'tmux kill-session -t game; tmux new-session -d -s game "cd /root/server/game
  && /root/venvs/game/bin/uvicorn app.main:app --host 0.0.0.0 --port 8080 2>&1 | tee /root/game.log"'`
  (camp-3는 몰캠 네트워크/VPN에서만 SSH 접속됨.)
